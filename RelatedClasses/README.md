# Projeto Educacional: Sistema de Pedidos em Java

## 1. Introdução

Este projeto tem como objetivo apoiar o aprendizado de **Programação Orientada a Objetos (POO)** utilizando a linguagem **Java**. A atividade foi desenvolvida para ser realizada em grupo, permitindo que os alunos pratiquem conceitos fundamentais da programação enquanto colaboram em um repositório compartilhado.

Durante o desenvolvimento do projeto, os estudantes terão contato com práticas comuns no desenvolvimento de software, como **versionamento de código**, **trabalho colaborativo** e **organização de um sistema orientado a objetos**.

O sistema proposto simula um **sistema simples de pedidos para uma cafeteria**, permitindo o cadastro de clientes, produtos e pedidos realizados.

---

## 2. Objetivos Educacionais

Esta atividade tem como finalidade permitir que os alunos desenvolvam habilidades relacionadas a:

- Programação Orientada a Objetos
- Criação e organização de classes
- Encapsulamento de dados
- Comunicação entre objetos
- Estruturação de programas com múltiplas classes
- Leitura e compreensão de código existente
- Trabalho colaborativo utilizando Git
- Realização de commits e envio de alterações para um repositório remoto

Além disso, os alunos terão a oportunidade de compreender como sistemas simples podem ser modelados a partir de entidades do mundo real.

---

## 3. Descrição do Sistema

O sistema representa um **ambiente simplificado de gerenciamento de pedidos de uma cafeteria**. O objetivo é modelar as principais entidades envolvidas no processo de compra de produtos.

As principais classes do sistema são:

- **Cliente**: representa a pessoa que realiza pedidos.
- **Produto**: representa os itens disponíveis para venda na cafeteria.
- **Pedido**: representa uma solicitação de compra feita por um cliente.
- **ItemPedido**: representa os produtos específicos incluídos em um pedido.

O relacionamento entre essas classes permite representar situações reais, como:

- Um cliente pode realizar vários pedidos.
- Um pedido pode conter vários produtos.
- Cada item de pedido representa um produto e sua quantidade.

---

## 4. Estrutura do Projeto

O projeto é composto por:

- Classes que representam as entidades do sistema
- Uma classe `Main` responsável pela execução do programa
- Um menu em terminal que permite a interação com o sistema

Os alunos deverão implementar funcionalidades relacionadas às opções do menu, como:

- Cadastro de clientes
- Cadastro de produtos
- Criação de pedidos
- Adição de itens aos pedidos
- Exibição de informações cadastradas

---

## 5. Trabalho Colaborativo

Cada integrante do grupo deverá:

- Clonar o repositório do projeto
- Implementar funcionalidades específicas
- Realizar **commits individuais**
- Enviar as alterações para o repositório do grupo utilizando **push**

Essa prática simula um ambiente real de desenvolvimento de software, no qual vários desenvolvedores trabalham simultaneamente em um mesmo projeto.

---

## 6. Resultado Final Esperado

Ao final da atividade, espera-se que o sistema seja capaz de:

1. Executar a aplicação a partir da classe `Main`.
2. Exibir um menu no terminal para interação com o usuário.
3. Permitir o cadastro de clientes e produtos.
4. Permitir a criação de pedidos.
5. Permitir a adição de produtos aos pedidos.
6. Exibir informações sobre pedidos e seus respectivos itens.

O sistema final deverá demonstrar a **integração entre as classes do projeto**, evidenciando o uso correto dos princípios de programação orientada a objetos.

---

## 7. Desafio Bônus

Como atividade adicional, os alunos poderão refatorar partes do sistema que utilizam **arrays** para utilizar **estruturas de coleção da linguagem Java**, como listas dinâmicas.

Essa melhoria permitirá que o sistema se torne mais flexível e próximo das práticas utilizadas em sistemas profissionais.

---

## 8. Considerações Finais

Este projeto foi desenvolvido com foco educacional, buscando proporcionar uma experiência prática no desenvolvimento de software. A atividade incentiva não apenas o aprendizado técnico, mas também habilidades importantes para a carreira em computação, como colaboração, organização de código e uso de ferramentas de controle de versão.
